# AI-Factory: Replit Backend Master Prompt

## 🎯 Mission

You are building the **AI-Factory backend** — the brain, the nervous system, and the data layer of an autonomous AI research operating system. A separate frontend (built in Lovable) will consume everything you produce. Your job is zero UI, pure infrastructure: FastAPI servers, WebSocket telemetry streams, job orchestration, database models, agent logic, and the autonomous training loop.

**The contract is simple:** Lovable renders. You compute. Every API endpoint, WebSocket message type, and data schema defined here maps 1:1 to what Lovable expects to consume. Do not deviate from the interface contracts — they are the seam between the two systems.

## 🧠 Core Service Implementations

### Connection Manager (WebSocket)

```python
# api/websocket.py
from collections import defaultdict
from fastapi import WebSocket
import asyncio, json

class ConnectionManager:
    def __init__(self):
        self.active: set[WebSocket] = set()
        self.subscriptions: dict[str, set[WebSocket]] = defaultdict(set)

    async def connect(self, ws: WebSocket):
        await ws.accept()
        self.active.add(ws)

    def disconnect(self, ws: WebSocket):
        self.active.discard(ws)
        for subs in self.subscriptions.values():
            subs.discard(ws)

    async def broadcast(self, message: TelemetryMessage):
        """Broadcast to all connected clients."""
        payload = message.model_dump_json()
        dead = set()
        for ws in self.active:
            try:
                await ws.send_text(payload)
            except Exception:
                dead.add(ws)
        for ws in dead:
            self.disconnect(ws)

    async def broadcast_to_subscribers(self, topic: str, message: TelemetryMessage):
        """Broadcast only to clients subscribed to a specific topic (job_id, node_id, etc.)"""
        payload = message.model_dump_json()
        for ws in self.subscriptions.get(topic, set()):
            try:
                await ws.send_text(payload)
            except Exception:
                pass

manager = ConnectionManager()
```

class JobService:
    def __init__(self, db: AsyncSession, manager: ConnectionManager):
        self.db = db
        self.manager = manager

    async def create_job(self, request: JobCreateRequest) -> JobDetail:
        """
        1. Validate dataset exists and is in 'ready' state
        2. Validate base model is accessible
        3. Select target node via ClusterService.select_node()
        4. Create DB record with status='queued'
        5. Dispatch Celery task: training_tasks.run_training_job.delay(job_id)
        6. Register in lineage registry
        7. Return JobDetail
        """

    async def stop_job(self, job_id: str) -> None:
        """
        1. Fetch job, assert status is 'running' or 'queued'
        2. Revoke Celery task (terminate=True)
        3. Send SIGTERM to training process on node
        4. Update status to 'stopped'
        5. Broadcast JobFailedMessage with error='User requested stop'
        6. Trigger checkpoint save if mid-run
        """

    async def update_progress(self, job_id: str, step: int, loss: float) -> None:
        """
        Called by training worker via Celery task update.
        1. Update DB record
        2. Append to loss_history (keep last 500 points)
        3. Calculate ETA from step velocity
        4. Broadcast JobUpdateMessage via WebSocket
        5. Notify AgentService for monitoring (non-blocking)
        """

    async def complete_job(self, job_id: str, checkpoint_path: str) -> str:
        """
        1. Update status to 'completed'
        2. Trigger eval_tasks.run_evaluation.delay(job_id, checkpoint_path)
        3. Register checkpoint with ModelService
        4. Broadcast JobCompleteMessage
        5. Notify AgentService.on_job_complete() for promotion decisions
        6. Return model_id
        """

### Cluster Service

```python
# services/cluster_service.py

class ClusterService:
    def __init__(self, db: AsyncSession, manager: ConnectionManager):
        self.db = db
        self.manager = manager
        self._drivers: dict[str, NodeDriver] = {}

    async def poll_all_nodes(self) -> None:
        """
        Beat task runs this every 10s.
        For each registered node:
          1. Call driver.get_health() — catches exceptions, marks node degraded/offline
          2. Update DB NodeStatus record
          3. Apply threshold rules:
             - VRAM > 95% → status = 'degraded', broadcast WARN log line
             - Temp > 85°C → status = 'degraded', broadcast WARN
             - No response 3x → status = 'offline'
          4. Broadcast ClusterUpdateMessage with all node statuses
        """

    async def select_node(self, job: JobCreateRequest) -> str:
        """
        Selects optimal node for a job using:
        1. Filter: nodes with status='online'
        2. Filter: nodes with enough free VRAM for job's model
        3. Score remaining nodes by:
           - Active job count (fewer = better)
           - VRAM headroom (more = better)
           - Job type affinity (apple silicon prefers smaller models)
        4. Return node_id of winner
        5. Raise ClusterError if no eligible nodes
        """

    async def dispatch_to_node(self, node_id: str, job_id: str) -> None:
        """
        1. Get driver for node
        2. For local nodes: Celery task runs in same process space
        3. For SSH remote nodes: driver.submit_job() — SSH + run training script
        4. Update job.node_id in DB
        """
```

### Agent Orchestrator

```python
# agents/orchestrator.py

class AgentOrchestrator:
    """
    Runs on a Celery beat schedule (every 30s).
    Coordinates the three agents: Evaluator, Pruner, Promoter.
    All decisions are logged to DB and broadcast via WebSocket.
    """

    def __init__(
        self,
        evaluator: EvaluatorAgent,
        pruner: PrunerAgent,
        promoter: PromoterAgent,
        job_service: JobService,
        manager: ConnectionManager,
    ):
        ...

    async def tick(self) -> None:
        """Main loop — called every 30s by Celery beat."""
        running_jobs = await self.job_service.get_running_jobs()

        # Pruner runs first — kill bad jobs before evaluating
        for job in running_jobs:
            prune_decision = await self.pruner.evaluate(job)
            if prune_decision.should_prune:
                await self.job_service.stop_job(job.id)
                await self._log_decision(prune_decision)

        # Evaluator scores completed jobs awaiting scoring
        unscoред_jobs = await self.job_service.get_completed_unscored()
        for job in unscored_jobs:
            eval_result = await self.evaluator.score(job)
            await self._log_decision(eval_result)

        # Promoter checks scored jobs against promotion threshold
        promotion_candidates = await self.job_service.get_promotion_candidates()
        for candidate in promotion_candidates:
            promotion = await self.promoter.evaluate(candidate)
            if promotion.should_promote:
                await self._escalate_to_dpo(candidate, promotion)
                await self._log_decision(promotion)

    async def _log_decision(self, decision: AgentDecision) -> None:
        """Save to DB + broadcast AgentDecisionMessage via WebSocket."""
        await self.db.add(decision)
        await self.manager.broadcast(AgentDecisionMessage(
            type="agent_decision",
            agent_id=decision.agent_id,
            agent_type=decision.agent_type,
            action=decision.action,
            target_id=decision.target_id,
            reasoning=decision.reasoning,
            timestamp=datetime.utcnow().timestamp(),
        ))
```

### Pruner Agent

```python
# agents/pruner.py

class PrunerAgent:
    """
    Implements ASHA (Asynchronous Successive Halving Algorithm).
    Kills training runs that are unlikely to converge.
    """

    PLATEAU_WINDOW = 500          # steps
    PLATEAU_THRESHOLD = 0.001     # minimum Δloss to not be considered plateaued
    DIVERGENCE_THRESHOLD = 5.0    # loss > 5x median of cohort → diverged
    MIN_STEPS_BEFORE_PRUNE = 500  # never prune before 500 steps

    async def evaluate(self, job: JobDetail) -> PruneDecision:
        if job.current_step < self.MIN_STEPS_BEFORE_PRUNE:
            return PruneDecision(should_prune=False)

        # Check loss plateau
        recent_losses = job.loss_history[-self.PLATEAU_WINDOW:]
        if len(recent_losses) >= self.PLATEAU_WINDOW:
            delta = max(recent_losses) - min(recent_losses)
            if delta < self.PLATEAU_THRESHOLD:
                return PruneDecision(
                    should_prune=True,
                    reason=f"Loss plateau: Δloss={delta:.5f} over last {self.PLATEAU_WINDOW} steps",
                )

        # Check divergence vs cohort median
        cohort_losses = await self._get_cohort_median_loss(job)
        if cohort_losses and job.current_loss > self.DIVERGENCE_THRESHOLD * cohort_losses:
            return PruneDecision(
                should_prune=True,
                reason=f"Divergence: loss {job.current_loss:.4f} > {self.DIVERGENCE_THRESHOLD}x cohort median {cohort_losses:.4f}",
            )

        return PruneDecision(should_prune=False)
```

## 🔗 Lovable Integration Checklist

These are the exact things Lovable depends on being correct. Verify each one before declaring the backend ready.

### CORS Configuration
```python
# main.py — Lovable dev server runs on localhost:3000
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "https://*.lovable.app"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### WebSocket CORS
The WebSocket endpoint at `/ws/telemetry` must accept connections from `localhost:3000` and `*.lovable.app`. FastAPI WebSocket does not use the CORS middleware — handle this manually in the endpoint.

### JSON Field Names
Lovable uses **camelCase** in TypeScript. All Pydantic models must serialize to **snake_case** (FastAPI default). Lovable's API client handles the conversion via a response interceptor — **do not change field names to camelCase on the backend.**

### Datetime Format
All `datetime` fields must serialize as **ISO 8601 with UTC timezone**: `"2024-11-15T14:32:01.123Z"`. Set `model_config = ConfigDict(json_encoders={datetime: lambda v: v.isoformat() + "Z"})` on all schemas.

### Null vs Missing Fields
Lovable checks for `null` explicitly on nullable fields (confidence_score, eta_seconds, parent_model_id, etc.). Never omit these fields from the response — always include them as `null`. Set `model_config = ConfigDict(populate_by_name=True)` and avoid `exclude_none=True` in responses.

### WebSocket Reconnection
Lovable implements automatic WebSocket reconnection with exponential backoff. The backend must handle rapid reconnections gracefully (no state corruption on disconnect/reconnect cycles). The ConnectionManager's `broadcast()` method already handles this by catching send exceptions.
